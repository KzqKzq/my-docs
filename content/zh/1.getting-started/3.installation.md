## `create-docus` CLI

::steps
### 创建文档目录

使用 `create-docus` CLI 创建新的 Docus 项目：

```bash [Terminal]
npx create-docus my-docs
```

您可以在两个模板之间选择：
- **`default`**：用于单语言文档的基本 Docus 设置
- **`i18n`**：包括对多语言文档的国际化支持

```bash [Terminal]
# 使用 i18n 模板创建
npx create-docus my-docs -t i18n
```

我们建议使用 `npm` 包管理器。

### 在开发中启动文档服务器

转到文档目录并以开发模式启动文档服务器：

```bash [Terminal]
cd my-docs
npm run dev
```

您的文档的本地预览将在 <http://localhost:3000> 处可用

### 编写文档

前往 [编辑](/zh/concepts/edition) 部分了解如何编写文档。
::

## 层集成

Docus v4 使用**基于 Nuxt 层的方法**，您可以在 `nuxt.config.ts` 中使用 `extends: ['docus']` 直接扩展 Docus 层：

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['docus']
})
```

## 使用 Nuxt Studio 进行在线编辑

::prose-steps
### 在 Nuxt Studio 上创建新项目

从您的用户或团队仪表板中，单击"新建项目"按钮，然后选择"从模板开始"并选择 **Docus**。将其克隆到您的 GitHub 个人账户或任何您选择的组织。

### 一键部署

项目创建后，当您进入项目仪表板时，导航至"部署"部分，选择"GitHub Pages"选项卡，然后单击"部署"按钮。

  :::prose-note
  ---
  to: https://content.nuxt.com/docs/studio/setup#enable-the-full-editing-experience
  ---
  这是一个通过 [GitHub Pages](https://docs.github.com/en/pages/getting-started-with-github-pages/creating-a-github-pages-site) 提供的一键静态部署，但您也可以自己处理部署并使用"自我托管"选项卡。
  :::

### 在编辑器中编写文档

部署完成后，您将能够查看文档的预览。您可以浏览内容页面进行编辑或创建新页面。
