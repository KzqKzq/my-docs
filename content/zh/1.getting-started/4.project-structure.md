## 全局结构

Docus v4 是一个 **Nuxt 层**，用 Docus 文档功能扩展您的标准 Nuxt 应用程序。这为您提供了完整 Nuxt 项目的灵活性。

当您使用 `npx create-docus my-docs` 创建新的 Docus 项目时，您会得到：

```
my-docs/
├── content/             # 您的 markdown 内容
│   ├── index.md         # 主页
│   └── docs/            # 文档页面
├── public/              # 静态资源
└── package.json         # 依赖和脚本
```

您仍然可以使用经典 Nuxt 项目的任何功能或文件：

```
my-docs/
├── app.config.ts        # 应用配置
├── nuxt.config.ts       # Nuxt 配置（添加额外模块、组件等）
├── app/                 # 应用目录
│   ├── components/      # 组件（添加您自己的组件）
│   ├── layouts/         # 布局（添加您自己的布局）
│   └── pages/           # 页面（添加您自己的页面）
└── server/              # 服务器端代码（添加您自己的服务器端代码）
```

### `content/` 目录

这是您[编写页面](/zh/concepts/edition)的地方，采用 Markdown 格式。Docus 根据您的文件结构自动生成路由。

**单语言结构：**
```
content/
├── index.md
├── getting-started.md
└── guide/
    ├── introduction.md
    └── configuration.md
```

**多语言结构（带 i18n）：**
```
content/
├── zh/
│   ├── index.md
│   └── guide/
│       └── introduction.md
└── en/
    ├── index.md
    └── guide/
        └── introduction.md
```

::prose-tip
有关 i18n 的更多信息，请访问[国际化](/zh/concepts/internationalization)部分。
::

### `public/` 目录

`public/` 目录中包含的文件在根目录处提供，并且在构建过程中不会被修改。这是您可以放置图像、图标和其他静态资源的地方。

### `package.json`

此文件包含应用程序的所有依赖项和脚本。Docus 应用程序的 `package.json` 非常最小化，看起来像这样：

```json [package.json]
{
  "name": "my-docs",
  "scripts": {
    "build": "nuxt build --extend docus",
    "dev": "nuxt dev --extend docus",
  },
  "dependencies": {
    "docus": "latest",
    "better-sqlite3": "^12.2.0",
    "nuxt": "^4.0.0"
  }
}
```

### `nuxt.config.ts`

*此文件不是启动 Docus 应用程序的必需文件。*

您可以在 Nuxt 配置文件中添加额外模块：

```typescript [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['docus']
})
```
