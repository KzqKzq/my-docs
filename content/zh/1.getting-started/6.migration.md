## **从 Docus v3 迁移到 v4**

Docus v4 引入了一种新的**基于层的方法**，利用官方 Nuxt CLI 而不是自定义 Docus CLI。虽然您现有的内容和配置仍然兼容，但您需要更新命令和项目设置。

### **⚠️ 破坏性更改**

主要的破坏性更改与 CLI 命令相关：

| v3 | v4 |
|--------------|----| 
| `npx docus init my-docs` | `npx create-docus my-docs` |
| `docus dev` | `nuxt dev --extend docus` |
| `docus build` | `nuxt build --extend docus` |

::prose-tip
您现有的 Markdown 内容和 MDC 语法将不需要更改即可工作。迁移主要涉及更新您的开发和构建工作流。
::

## **迁移到 Docus**

已经在使用 Markdown 的文档解决方案？无论是 **Docus v1**、**Nuxt UI docs 模板**还是另一个静态网站设置，迁移到 Docus 都很简单直接。

Docus 提供了一个干净可维护的解决方案，仅依赖一个依赖项：Docus 库本身。无需管理多个依赖项。由于所有内容都是内置的并一起维护，保持文档的更新变得更加容易。

要迁移，只需将现有的 Markdown 文件移动到 Docus 启动程序的 `content/` 目录中。

从那里，您有两种情况：

- **如果您当前的文档已经使用 Nuxt Content 和 MDC 语法**，请确保您的内容中使用的组件存在于 Nuxt UI 中。如果缺少任何组件，您可以轻松创建自己的自定义组件。
- **如果您使用标准 Markdown**，您可以按原样复制文件。然后，使用 [内置组件](/zh/essentials/components)逐步增强您的文档，该组件由 Nuxt UI 提供。

将内容移动到 `content/` 文件夹后，您可以查看[配置部分](/zh/concepts/configuration)以轻松自定义您的应用。

Docus 的设计重点是编写内容，因此如果您已经在使用 Markdown，您可以轻松切换到它。
